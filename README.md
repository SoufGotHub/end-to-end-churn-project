# Churn Prediction Project

An end-to-end machine learning project designed to predict telecom customer churn. This repository contains a complete pipeline that generates realistic synthetic data (simulating customer demographics, billing history, support interactions, and usage patterns) and trains a Logistic Regression model to identify customers at risk of leaving.

## Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/your-username/your-repository.git
   ```
2. Install the dependencies:
   ```bash
   pip install -r requirements.txt
   ```

## Data

The pipeline uses a generated dataset located in `data/raw`:

- `dataset_churn_simule.csv`: Simulated dataset generated by `scripts/01_generate_dataset.py` and used by `scripts/02_train_model.py`.

To regenerate the dataset:

```bash
python scripts/01_generate_dataset.py
```

## Scripts

The `scripts` directory contains the following Python scripts:

- `01_generate_dataset.py`: Generates a simulated dataset for churn prediction.
- `02_train_model.py`: Trains a logistic regression model on the generated dataset and saves the trained model to the `models` directory.

## Models

The `models` directory contains the trained model:

- `churn_logistic_regression.joblib`: A logistic regression model trained to predict customer churn. This model is generated by the `02_train_model.py` script.

## Docker

The `Dockerfile` in the root directory can be used to build a Docker image for this project.

To build the image, run the following command:

```bash
docker build -t churn-prediction .
```

To run the container, use the following command:

```bash
docker run -it churn-prediction
```

## SQL

The `sql` directory contains the following SQL scripts:

- `schema.sql`: Defines the database schema for the project.
- `query_extraction_features.sql`: Contains queries for extracting features from the database.
